250 /* Projection of a simple volume ------------------------------------------- */
251 // The projection is not precleaned (set to 0) before projecting and its
252 // angles are supposed to be already written (and all Euler matrices
253 // precalculated
254 // The projection plane is supposed to pass through the Universal coordinate
255 // origin
256 
257 /* Time measures ...........................................................
258  This function has been optimized in time, several approaches have been
259  tried and here are the improvements in time
260  Case: Base (unit time measure)
261  The points within the footprint are calculated inside the
262  inner loop instead of computing the foot coordinate for the
263  first corner and knowing the sampling rate in the oversampled
264  image go on the next points. The image access was done directly
265  (physical access).
266 
267  Notation simplification:
268  Case: VOLVOXEL(V,k,i,j) ----> V(k,i,j): + 38% (Inacceptable)
269  Case: DIRECT_IMGPIXEL ----> IMGPIXEL: + 5% (Acceptable)
270 
271  Algorithmic changes:
272  Case: project each basis position ----> +325% (Inacceptable)
273  get rid of all beginZ, beginY, prjX, ... and project each
274  basis position onto the projection plane
275  Case: footprint coordinates outside ----> - 33% (Great!!)
276  the footprint coordinates are computed outside the inner
277  loop, but you have to use the sampling rate instead to
278  move over the oversampled image.
279 */
