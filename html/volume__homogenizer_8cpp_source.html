64  addParamsLine(" [--pyr_scale <pyr_scale=0.5>] : parameter, specifying the image scale (<1) to build pyramids for each image.");
65  addParamsLine(" : pyr_scale=0.5 means a classical pyramid, where each next layer is twice smaller than the previous one.");

307  /*OF algorithm/*
308  Parameters:
309  ----------
310  prev first 8-bit single-channel input image.
311  next second input image of the same size and the same type as prev.
312  flow computed flow image that has the same size as prev and type CV_32FC2.
313  pyr_scale parameter, specifying the image scale (<1) to build pyramids for each image; pyr_scale=0.5 means a classical pyramid, where each next layer is twice smaller than the previous one.
314  levels number of pyramid layers including the initial image; levels=1 means that no extra layers are created and only the original images are used.
315  winsize averaging window size; larger values increase the algorithm robustness to image noise and give more chances for fast motion detection, but yield more blurred motion field.
316  iterations number of iterations the algorithm does at each pyramid level.
317  poly_n size of the pixel neighborhood used to find polynomial expansion in each pixel; larger values mean that the image will be approximated with smoother surfaces, yielding more robust algorithm and more blurred motion field, typically poly_n =5 or 7.
318  poly_sigma standard deviation of the Gaussian that is used to smooth derivatives used as a basis for the polynomial expansion; for poly_n=5, you can set poly_sigma=1.1, for poly_n=7, a good value would be poly_sigma=1.5.
319  flags
320  */
